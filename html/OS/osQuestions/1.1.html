  • 处理外部中断时，应该由操作系统负责保存的有

    • 答案
      • 程序计数器中的内容
      • 通用寄存器中的内容（√）
      • 快表（TLB）的内容
      • Cache中的内容
    • 解析
      • 处理外部中断首先要保护中断现场。1 由中断隐指令保存断点， 即程序计数器（pc）中的内容；2 由操作系统的中断服务程序保存通用寄存器和状态寄存器的内容

  • 执行下列已经装入指令寄存器，则执行时不可能导致CPU从用户态变成内核态的是：

    • 答案 c
      • DIV R0,R1; (R0)/(R1)->R0//除数为0导致异常
      • INT n; 产生软中断//在内核态中执行
      • NOT R0；把寄存器中的内容取非；
      • MOV R0,addr;把地址addr处的内存放进寄存器R0中//可能导致地址越界异常
    • 解析
      • 系统调用、中断、异常都在内核态进行处理，凡是会设计到这三个的都会导致进入内核态

  • 内部异常（内中断）可以分为故障（fault）、陷阱（trap）和终止（abort）三类。下列有关异常的叙述中错误的是

    • 答案 d
      • 内部异常的产生与当前执行指令相关
      • 内部异常的检测由cpu内部逻辑实现
      • 内部异常的响应发生在指令执行过程中
      • 内部异常处理后返回到发生异常的指令继续执行
    • 解析
      • 外部中断才会返回指令继续执行，异常会结束任务

  • 中断程序和字处理程序调用都需要压栈以保护现场，中断处理一定会保存而子程序调用不需要保存的内容是

    • 答案 b
      • 程序计数器
      • 程序状态字寄存器
      • 通用数据寄存器
      • 通用地址寄存器
    • 解析
      • 子程序调用只需要保存程序断点，即该指令的下一条指令地址；中断处理不但要保存断点，还要保存程序状态寄存器的内容。

  • 响应外部中断的过程中，中断隐指令完成的操作，除了保护断点外，还包括：

    • 答案 1,3
      • 关中断
      • 保存通用寄存器的内容
      • 形成中断服务程序入口并送PC
    • 解析
      • 响应外部中断的时候，中断隐指令的操作包括关中断、保护断点、引出中断服务程序；保护通用寄存器内容是OS的工作，不是中断隐指令

  • 若一个用户进程通过调用read系统调用读取一个磁盘文件的数据，则下列叙述正确的是

    • 答案 1,2
      • 若该文件数据不在内存，则该进程进入睡眠状态
      • 请求read系统调用会导致CPU从用户态进入内核态。
      • read系统调用的参数有应该包含文件名称
    • 解析
      • 读文件时先用open系统调用打开文件，open的参数包含文件路径和文件名，read只需要使用open返回的文件描述符，不用再使用文件名。

  • 引入多道程序技术的前提条件之一是系统具有

    • 答案 b

    • 解析

  • 计算机系统中有些操作必须使用特权指令完成，下面（）操作不需要使用特权指令

    • 答案 c
      • 修改界限地址寄存器
      • 设置定时器初值
      • 触发访管指令
      • 关闭中断允许位
    • 解析
      • 特权指令是指在内核态完成的指令，只允许OS使用，不允许应用程序使用。访管指令操作不需要使用特权指令。
      • 界地址寄存器：
        • 在CPU中设置一对下限寄存器和上限寄存器,存放用户作业在主存中的下限和上限地址；
        • 可将一个寄存器作为基址寄存器，另一寄存器作为限长寄存器（指示存储区长度）；
        • 每当CPU要访问主存，硬件自动将被访问的主存地址与界限寄存器的内容进行比较，以判断是否越界；
        • 如果未越界，则按此地址访问主存，否则将产生程序中断——越界中断（存储保护中断）
      • 允许中断是指处理机在处理当前中断或其他进程时，允许响应来自外部中断。
